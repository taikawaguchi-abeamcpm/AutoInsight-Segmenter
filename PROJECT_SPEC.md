# プロジェクト名: AutoInsight & Segmenter

## 1. サービスコンセプト
ビジネスユーザーがデータサイエンスの知識なしに、Microsoft Fabric上のデータから「成功の黄金パターン」を発見し、即座にマーケティング施策（セグメント作成）へ繋げるためのノーコード・インサイト・プラットフォーム。

## 2. システムアーキテクチャ
- **Frontend**: Azure Static Web Apps (React + TypeScript + Tailwind CSS)
- **Data Backend**: Microsoft Fabric (GraphQL API経由でのデータ取得)
- **Infrastructure**: Azure (Identity, Static Web Apps)

## 3. 主要機能
### A. セマンティック・マッピング
- Fabricから取得したテーブル・カラムに対し、「顧客マスタ」「購買履歴」といったビジネス的意味付けを行う。
- 目的変数（例：成約）と特徴量（例：Web閲覧履歴）をユーザーが定義する。

### B. カスタム分析（実験開始）
- 指定された目的変数に対する特徴量の影響度を分析。
- 「黄金パターン（例：Web閲覧＋架電＝成約率UP）」を抽出し可視化する。

### C. オートパイロット（自動実験開始）
- 目的変数のみ指定。AIが自動で特徴量生成（エンジニアリング）と最適なパターン抽出を行う。

### D. セグメンテーション
- 分析結果に基づき、条件に合致する顧客にフラグを付与し、リスト化する。

## 4. AIエージェントへのコマンド定義
エージェントは以下のコマンドを受け取った際、定義されたステップで実行すること。

### コマンド①: /design (設計書作成)
- **実行内容**: 
  1. 画面遷移図および各画面のUIコンポーネント設計の作成。
  2. Microsoft Fabric GraphQLとのデータスキーマ定義。
  3. 分析ロジック（影響度算出やパターン抽出）のアルゴリズム案の策定。
- **アウトプット**: `docs/design/` 配下にMarkdown形式で保存。

### コマンド②: /implement (プログラム作成)
- **実行内容**:
  1. 設計書に基づいたコードの実装（Frontend/Backend）。
  2. TypeScriptを用いた型安全な実装。
  3. Azure Static Web Appsへのデプロイを考慮した構成ファイルの作成。
- **アウトプット**: `src/` 配下へのコード生成。

### コマンド③: /review (プログラムレビュー)
- **実行内容**:
  1. 実装されたコードのバグ、セキュリティ脆弱性、パフォーマンスのチェック。
  2. 特に「Fabric API呼び出しの効率性」と「UIのアクセシビリティ」を重点的に確認。
  3. 修正案の提示。
- **アウトプット**: `docs/reviews/` 配下またはチャット上でのフィードバック。

### コマンド④: /review_design (設計書総合レビュー)
- **実行内容**:
  1. `docs/design/` 配下にあるすべての設計書（画面定義、データスキーマ、ロジック設計など）を横断的に読み込み、矛盾や考慮漏れがないか確認する。
  2. プロジェクトのコアバリュー（ビジネスユーザー向けのノーコード分析）や、Azure/Fabricのアーキテクチャ制約に沿っているかを評価する。
  3. エラーハンドリング、エッジケース、セキュリティの観点から不足している仕様を洗い出す。
- **アウトプット**: `docs/reviews/` 配下に `design_review_report.md` などのファイル名で、改善提案を含む総合レビューレポートを出力する。

### コマンド⑤: /update_design (設計書修正)
- **実行内容**:
  1. `docs/reviews/` 配下にある設計レビュー結果（`design_review_report.md` など）を読み込む。
  2. レビューで指摘された矛盾点、考慮漏れ、改善案に基づき、`docs/design/` 配下の該当する設計書を修正・アップデートする。
  3. 修正の意図や反映状況が人間にも分かるよう、修正内容のサマリー（変更履歴）を提示する。
- **アウトプット**: 更新された `docs/design/` 配下のMarkdownファイル群。

### コマンド⑥: /update_code (プログラム修正)
- **実行内容**:
  1. `docs/reviews/` 配下のコードレビュー結果、またはチャット上で指摘された改善事項を読み込む。
  2. レビューで指摘されたバグ、脆弱性、パフォーマンスの課題、または設計書との乖離に基づき、`src/` 配下の該当ソースコードを修正・リファクタリングする。
  3. 修正箇所とその理由が人間にも分かるよう、変更内容のサマリーを提示する。
- **アウトプット**: 更新された `src/` 配下のソースコード群。
