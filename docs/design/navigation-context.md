# 画面遷移・コンテキスト受け渡し 設計書

## 1. 目的
画面間で渡すID、ルーティングstate、一時情報、API再取得対象を整理し、ブラウザリロード、戻る/進む、別タブ表示に耐える導線にする。

## 2. 基本方針
- URLには永続リソースIDのみ持たせる。
- 表示に必要な詳細情報はAPIから再取得する。
- ルーティングstateは UX 補助に限定し、リロードで失われても復元できるようにする。

## 3. 画面別URLパラメータ
| 画面 | URLに保持するID | API再取得対象 |
| --- | --- | --- |
| Fabric 接続管理 | なし、または `connectionId` | 保存済み接続設定、疎通状態 |
| データセット選択 | なし | ワークスペース/データセット一覧 |
| セマンティック・マッピング | `datasetId` | Fabricスキーマ、マッピング下書き |
| 分析条件確認 | `mappingDocumentId` | 分析入力サマリ、分析下書き |
| 結果可視化 | `analysisJobId` | ジョブ状態、結果 |
| セグメント作成 | `analysisJobId`, `segmentIds` | セグメントドラフト、候補詳細 |

## 4. 受け渡しモデル
- `SelectedDatasetContext` は遷移直後の表示高速化に使うが、画面初期化では `datasetId` から再取得する。
- `SelectedSegmentContext` は候補選択の初期値として使い、保存済みドラフトがある場合はサーバー状態を優先する。
- ブラウザ戻る時は未保存変更がある場合のみ確認ダイアログを出す。

## 5. 期限切れ・権限変更
- URL上のIDが存在しない場合は「見つかりません」表示にする。
- 権限が失われた場合は `ACCESS_DENIED` とし、データ内容は表示しない。
- 実行済みジョブが削除/アーカイブ済みの場合は、結果一覧またはデータセット選択へ戻る導線を出す。
